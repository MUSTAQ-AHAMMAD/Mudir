# مدير · Mudir — منسّق المشاريع بالذكاء الاصطناعي عبر واتساب

> 🌐 هذه ترجمة عربية لملف [`README.md`](../../README.md). عند وجود اختلاف، يُعتمد الملف الإنجليزي كمرجع.

**مدير** هو منسّق مشاريع B2B يعمل بالذكاء الاصطناعي ويعيش **داخل واتساب**. ينسّق
المشاريع متعددة الفرق مثل افتتاح المتاجر: تُكمل الفرق (العقارات ← التسويق ←
تقنية المعلومات) المهام بالتسلسل، بينما يتتبّع البوت التقدّم، ويرسل التذكيرات،
ويصعّد التأخيرات، ويؤكّد افتتاح المتجر في موعده.

> مبني لسلاسل التجزئة السعودية. جميع ردود البوت **ثنائية اللغة** — العربية أولًا
> ثم الإنجليزية — ويحترم أسبوع العمل السعودي (الأحد–الخميس، مع تجاوز الجمعة).

---

## ✨ المميزات

- 📱 **يعمل داخل واتساب** — ينسّق قادة الفرق بالكامل من المحادثة (مجموعة أو رسالة مباشرة)
- 🧠 **يتعلّم مسارات العمل ديناميكيًا** — يستنتج الذكاء الاصطناعي مراحل المشروع من المحادثة، ويعمل لأي قطاع (تجزئة، برمجيات، فعاليات، إنشاءات…)
- 🎙️ **الرسائل الصوتية** — تفريغ الكلام العربي/الإنجليزي محليًا (Whisper) وتحويله إلى إجراءات
- 🖼️ **استقبال الصور والمستندات** — يحوّل التعرّف الضوئي على الحروف (OCR) التصاريح/الإيصالات إلى تحديثات منظّمة
- 🔀 **تنسيق قائم على الحالة** — آلة حالة تراعي الاعتماديات لتقديم المراحل واكتشاف العوائق وحساب التقدّم
- ⏰ **توقّع التأخير والتصعيد التلقائي** — أي تأخير ≥ 3 أيام يُصعَّد تلقائيًا للرئيس التنفيذي
- 🌍 **ثنائي اللغة والعربية أولًا** — كل رد بالعربية أولًا ثم الإنجليزية
- 🇸🇦 **جدولة تراعي السعودية** — تحترم أسبوع العمل (الأحد–الخميس) وتتجاوز الجمعة
- 🔒 **ذاتي الاستضافة 100%** — Ollama و Whisper و BGE-M3 و ChromaDB و NLLB؛ **بلا OpenAI وبلا تكلفة لكل رسالة**، وتبقى بياناتك على خادمك
- 📊 **لوحة تحكم إدارية** — React 18 + Vite + Tailwind، دعم RTL، الوضع الداكن، تحليلات
- 🚀 **جاهز للإنتاج** — Docker Compose (GPU/CPU)، nginx مع TLS، مراقبة، نسخ احتياطي، CI/CD

## 🏗️ البنية المعمارية

يتكوّن النظام من ثلاث طبقات:

- **الواجهة الخلفية** (`backend/`, Node.js/Express) — الويب هوك، ومحلّل الأوامر،
  وواجهة REST للوحة التحكم، والمهام المجدولة.
- **محرّك ORCHESTRA** (`orchestra/`, Python) — دماغ التنسيق: خدمات الذكاء
  الاصطناعي الذاتية، وطبقة قاعدة البيانات غير المتزامنة، ومحرّك التنسيق، وطبقة
  تكامل واتساب.
- **الواجهة الأمامية** (`frontend/`, React) — لوحة التحكم الإدارية.

مخطّطات البنية وتدفّق البيانات وتسلسل العمليات موجودة في
[`docs/architecture.mermaid`](../architecture.mermaid).

## 🚀 البدء السريع

**المتطلبات:** Node.js ≥ 18، ومشروع Supabase، ومرسل واتساب عبر Twilio/WATI،
وحزمة الذكاء الاصطناعي الذاتية (تُثبَّت عبر `auto-install.sh`).

```bash
# الواجهة الخلفية
cd backend
cp .env.example .env      # املأ TWILIO_* و SUPABASE_* و DATABASE_URL
npm install
npm test
npm run dev               # http://localhost:3000

# الواجهة الأمامية
cd ../frontend
cp .env.example .env
npm install
npm run dev               # http://localhost:5173
```

وجّه ويب هوك واتساب إلى `POST https://<العنوان-العام>/webhook`.

## 💬 أوامر واتساب

| الأمر | الوظيفة |
| --- | --- |
| `/new_project [الاسم]` | إنشاء مشروع وبدء المرحلة الأولى |
| `/assign [الفريق] [المهمة] [الموعد]` | إسناد مهمة (الموعد `YYYY-MM-DD`) |
| `/complete [معرّف المشروع]` | إنهاء المرحلة الحالية والانتقال للفريق التالي |
| `/extend [معرّف المشروع] [الفريق] [الأيام]` | تمديد موعد فريق (يُنبّه الرئيس التنفيذي) |
| `/status [معرّف المشروع]` | تقرير كامل عن الجدول الزمني للمشروع |
| `/escalate [معرّف المشروع] [السبب]` | تنبيه عاجل لجهة التصعيد |
| `/help` | عرض الأوامر |

اللغة الطبيعية مدعومة أيضًا — لست مضطرًا لحفظ الأوامر. الرسائل الصوتية تُفرَّغ
عبر Whisper، وعبارات مثل «خلصنا / we're done» تُربَط بأمر الإكمال **بعد التأكيد**.

## 🧪 الاختبارات

يمتلك محرّك بايثون مجموعة اختبارات شاملة (وحدات + تكامل + شامل) تحاكي كل الخدمات
الخارجية، فتعمل في أي مكان دون GPU أو نماذج أو شبكة:

```bash
pip install -r requirements-test.txt
pytest                                   # المجموعة كاملة
pytest --cov=orchestra --cov-report=html # مع تغطية
pytest -m e2e                            # اختبارات المسار الكامل فقط
```

تُتخطّى اختبارات قاعدة البيانات ما لم يُضبط `ORCHESTRA_TEST_DATABASE_URL` على
قاعدة PostgreSQL حقيقية.
